- **2026-04-07 (WebSocket Dev Tunnel auth fix):** Fixed three "silent killers" preventing WebSocket connections through private Microsoft Dev Tunnels. (1) **Subprotocol auth** — Switched from `?token=<JWT>` query parameter to `access_token-<JWT>` WebSocket subprotocol. Private Dev Tunnels (and enterprise proxies) strip query params during WebSocket Upgrade; subprotocol survives relay. Ticket-exchange path still uses query param for short-lived tickets (by design). (2) **Trailing slash** — Dev Tunnel relay interprets trailing slashes as HTTP GET for directory listing, not WebSocket Upgrade. Added `config.wsUrl.trim().replace(/\/+$/, '')` before URL construction. (3) **Close code diagnostics** — `ws.onclose` now logs `event.code`, `event.reason`, and `event.wasClean`; `ws.onerror` now logs the error event. Previously these handlers were silent, making auth failures (4xx) indistinguishable from relay issues (1006).
