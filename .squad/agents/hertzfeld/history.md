- **2026-04-05:** Protocol constraints that need heavy edge-case coverage: 20 msg/min WS rate limit, 500-message replay buffer, 4-hour session TTL, one-time ticket auth. These are the production fire starters.
- **2026-04-05:** C64 40-column mode is a testing concern — need to verify line wrapping/truncation behavior when switching from 80-col Apple IIe to 40-col C64 mid-session.
- **2026-04-05 (Cross-Agent):** Scaffold ready for test implementation. Vitest config included (`vitest.config.ts`), path alias `@/` synchronized between vite and tsconfig, structure supports co-located tests. Woz's 74 files deliver foundation for 60+ test cases. Start writing tests against M0 scaffold immediately.
- **2026-04-05:** Implemented test suite — 151 tests across 6 test files (3 hooks, 2 components, 1 theme config). All passing. MockWebSocket and MockAudioContext utilities created in `src/__mocks__/`. Key finding: WebSocket `connect()` resets `retriesRef` to 0 on every call including reconnect — this means exponential backoff delay never actually escalates and maxRetries is never reached through the reconnect timer path. Filed as a known behavior issue for Woz.
- **2026-04-05:** Kare expanded themes from 2→5 mid-session. Tests needed immediate adjustment: toggle cycle is now 5-step, title text changed, labels record expanded. localStorage pollution between test files was a real issue — every component test file wrapping in ThemeProvider must `localStorage.clear()` in `beforeEach`. jsdom doesn't reliably convert hex colors in inline styles to rgb — use structural assertions (element presence) over computed style checks.
- **Wave 2 completion** (2026-04-05): All 151 tests passing. Comprehensive coverage of hooks, components, and theme configs. Critical bug identified: WebSocket reconnect backoff never escalates (connect() resets retry counter on every call). Bug assigned to Woz for M1. Test suite is stable and effective — caught real production risk. localStorage clearing pattern established, jsdom color conversion workaround documented.
- **Wave 3 completion** (2026-04-05): 233 tests across 10 files, all passing. Woz and Kare delivered ConnectionManager class, command parser, MechanicalSwitch, AudioToggle, and expanded useAudio (11 sound types with SoundSequence multi-step support). Key findings: (1) Woz correctly fixed the backoff bug — `connect()` no longer resets retries, only `onopen` does; `connectFresh()` added for user-initiated connects. (2) Audio expanded from 5→11 types with mute/toggleMute and localStorage persistence. (3) MechanicalSwitch uses props `{crtEnabled, onToggle}` not store — three visual variants (lever/checkbox/pill) per skin. (4) AudioToggle takes `{muted, onToggle}` props with useTheme styling. (5) Component tests live in `src/components/__tests__/` — import paths are `../ComponentDir/Component`, not `../../`. (6) ConnectionManager rate limiting uses threshold (16) + hard cap (20) with drain timer, not simple counter. (7) Commands module uses singleton `connectionManager` + `useConnectionStore.getState()` — no context param.
- **Wave 4+6 tests** (2026-04-05): 314 tests across 13 files (293 passing, 21 skipped). Added 3 new test files: (1) `src/components/__tests__/TelemetryDrawer.test.tsx` — 21 tests in 5 `describe.skip` blocks covering rendering, connection metrics, session info, /status endpoint, and edge cases. All skipped pending Woz's implementation. Tests cover: keyboard shortcut open/close, latency/msg-per-sec/uptime display, masked session token, auto-refresh with 30s interval, graceful error handling, rapid toggle resilience. (2) `src/styles/__tests__/fonts.test.ts` — 32 tests verifying @font-face declarations (font-display:swap on all 6 rules), theme fallback chains (each theme's primary+fallback+generic stack), and graceful degradation (custom font names match CSS declarations). Key finding: font stacks changed from original spec — Apple IIe uses "Apple II" fallback not "Courier New", C64 uses "PetMe", IBM uses "IBM Plex Mono", Win95 uses "Fixedsys"/"Courier New", LCARS uses "Antonio". (3) `src/components/__tests__/accessibility.test.tsx` — 28 tests covering keyboard navigation (Tab focus + Enter/Space activation for ThemeToggle, MechanicalSwitch, AudioToggle, StatusBar), ARIA attributes (aria-label, aria-pressed, aria-hidden, title attributes), and reduced motion (CRT flicker scoping, CRTOverlay conditional rendering, matchMedia mock for prefers-reduced-motion). Key pattern: Zustand store updates in tests must be wrapped in `act()` to avoid React warnings.
