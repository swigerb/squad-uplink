## Work Completed

### Root Cause Analysis

#### HMR Warning Origin
- **Error message:** "ThemeProvider is not exported from React" (Fast Refresh warning)
- **Cause:** Single file `src/hooks/useTheme.tsx` exported both a custom hook (`useTheme`) and a context provider component (`ThemeProvider`)
- **Why problematic:** React Fast Refresh treats files exporting components differently from hooks-only files. Mixing them confuses the HMR system, triggering false "not exported from React" warnings.
- **Solution:** Split into two files — hooks file (function exports only) and component file (React component exports)

### Refactoring Architecture

#### New File Structure
```
src/hooks/
  ├── useTheme.tsx         (hook + context creation)
  └── ThemeProvider.tsx    (component wrapper)
```

#### `src/hooks/useTheme.tsx` (After)
```typescript
import { createContext, useContext } from 'react';
import { TerminalTheme } from '@/themes/types';

// Context creation (hooks-only file)
export const ThemeContext = createContext<TerminalTheme | undefined>(undefined);

// Hook (hooks-only file)
export const useTheme = (): TerminalTheme => {
  const theme = useContext(ThemeContext);
  if (!theme) throw new Error('useTheme must be used within ThemeProvider');
  return theme;
};
```

- **Exports:** `ThemeContext`, `useTheme` (function only, no components)
- **Purpose:** Pure hooks file for HMR reliability
- **Usage:** Import with `import { useTheme, ThemeContext } from '@/hooks/useTheme'`

#### `src/hooks/ThemeProvider.tsx` (New)
```typescript
import { ReactNode } from 'react';
import { ThemeContext } from './useTheme';
import { TerminalTheme } from '@/themes/types';

interface ThemeProviderProps {
  children: ReactNode;
  theme: TerminalTheme;
}

export const ThemeProvider = ({ children, theme }: ThemeProviderProps) => (
  <ThemeContext.Provider value={theme}>
    {children}
  </ThemeContext.Provider>
);
```

- **Exports:** `ThemeProvider` (component only)
- **Purpose:** Separate component file for HMR handling
- **Usage:** Import with `import { ThemeProvider } from '@/hooks/ThemeProvider'`

### Technical Details

#### Why This Fixes HMR
- **React Fast Refresh heuristic:** Files exporting only functions (hooks) get different handling than files exporting React components
- **Mixed exports problem:** Single file with both hook + component confuses the HMR runtime about which type of update to perform
- **Solution:** Separate them into two files — Refresh correctly identifies useTheme.tsx as a hooks module and ThemeProvider.tsx as a component module
- **Benefit:** Each file can be HMR-reloaded using the appropriate strategy for its content type

#### Import Resolution
- **Path alias `@/` still works:** Both new files in `src/hooks/` are accessible via `@/hooks/useTheme` and `@/hooks/ThemeProvider`
- **No breaking changes:** Hook consumers see no difference; only component consumers need import update
- **TypeScript inference:** All type hints remain intact; no `type` vs `value` complexity
